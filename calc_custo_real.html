NF-e XML Automático

Importe o arquivo .xml da Nota Fiscal Eletrônica. O sistema detecta automaticamente se é compra de madeira ou adesivo pelo nome do produto e extrai fornecedor, data, valor e quantidade.

Como obter o XML: peça ao seu contador ou acesse o portal da Sefaz com o número da NF. Aceita múltiplos arquivos de uma vez.

Planilha — CSV / Excel / Google Sheets

Google Sheets: Arquivo → Fazer download → .xlsx ou .csv.
Excel: Salvar como .xlsx ou .csv.
Selecione o tipo de dado antes de abrir o arquivo.

Nomes de coluna reconhecidos automaticamente:
Madeira: fornecedor volume_m3 valor_total data_recebimento nf
Adesivo: fornecedor kg valor_total produto
Pedidos: cliente orcamento valor_venda data_entrega local
Itens de Pedido: orcamento cliente descricao qtd base_mm altura_mm comp_mm comprimento
PDF de NF Não automático

PDFs não têm estrutura de dados lida por máquina — são imagens de texto. Não é possível extrair dados automaticamente sem OCR.

Alternativas:
• Peça ao fornecedor ou contador o arquivo .xml da NF-e (mesmo número, formato diferente) — funciona perfeitamente com a opção acima.
• Ou preencha manualmente nas abas Madeira / Adesivo com os dados do PDF.

Banco de dados remoto Supabase/Postgres para vários locais

Execute o SQL em database/supabase.sql no Supabase. Depois preencha URL, anon key e token. O token fica salvo só neste navegador.

Salvar dados no GitHub Usa um arquivo JSON no repositório

Para gravar, crie um fine-grained token no GitHub com acesso a este repositório e permissão Contents: Read and write. O token fica salvo só neste navegador.
